# AutoAds GoFly 重构需求规格说明书

## 1. 项目概述

### 1.1 项目背景
AutoAds 是一个基于 Next.js 的自动化营销 SaaS 平台，提供三大核心功能：批量访问(BatchOpen)、网站排名(SiteRank)和广告管理(AdsCenter)。当前系统使用 Next.js API Routes 作为后端，面临性能瓶颈和扩展性限制。本项目旨在将后端迁移至 GoFly 框架，实现性能提升和更好的多用户支持。

### 1.2 重构目标
1. **性能提升**：利用 Go 语言并发优势，实现 4900% 性能提升（从 1 并发提升到 50 并发）
2. **架构升级**：从 Node.js 单体应用演进为 Go 单体应用+模块化设计
3. **多用户支持**：完善的多用户数据隔离和权限管理
4. **后台管理**：集成 GoFly 框架的专业后台管理系统
5. **业务连续性**：保持所有现有功能的完整性和前端界面不变

### 1.3 技术选型
- **后端框架**：GoFly Admin V3
- **编程语言**：Go 1.21+
- **数据库**：MySQL 8.0（无需数据迁移，使用新数据库）
- **缓存**：Redis 7.0
- **前端**：保持现有 Next.js + TypeScript 架构
- **认证**：JWT + OAuth2.0

## 2. 功能需求

### 2.1 用户认证系统
- 支持邮箱注册和验证
- 集成 Google OAuth2.0 一键登录
- 用户资料管理（头像、昵称等）
- 密码重置和安全设置
- JWT token 认证机制

### 2.2 管理员系统
- 独立的管理员登录入口（账号密码）
- 用户管理功能
- 系统配置管理
- 任务监控和日志查看
- 套餐和 token 管理

### 2.3 用户套餐与权限
- 三级套餐体系：Free/Pro/Max
- 基于套餐的功能权限控制
- Token 经济系统
- 新用户 14 天 Pro 试用
- 邀请奖励机制（各得 30 天 Pro）

### 2.4 BatchGo 模块（原 BatchOpen）
#### 功能要求
- 完整迁移三种执行模式：Basic/Silent/Automated
- HTTP 和 Puppeteer 两种访问方式
- 高并发任务处理（最多 50 并发）
- 代理配置和轮换机制
- 任务结果统计和分析

#### 权限控制
- Free：仅 Basic 模式（前端标签页打开）
- Pro：Basic + Silent 模式
- Max：所有模式（包括 Automated）

### 2.5 SiteRankGo 模块（原 SiteRank）
#### 功能要求
- 完整迁移 SimilarWeb API 集成
- 批量域名查询功能
- 智能缓存策略
- 历史数据对比和趋势分析

#### 查询限制
- Free：100 个域名/次
- Pro：500 个域名/次
- Max：5000 个域名/次

### 2.6 AdsCenterGo 模块（原 AdsCenter）
#### 功能要求
- Google Ads OAuth 完整集成
- 多广告账户管理
- 链接提取和批量替换功能
- 集成 AdsPower 浏览器自动化
- 任务执行状态实时监控

### 2.7 Token 经济系统
#### Token 消耗规则
- SiteRankGo：成功查询 1 域名 = 1 token
- BatchGo HTTP 模式：成功访问 1 URL = 1 token
- BatchGo Puppeteer 模式：成功访问 1 URL = 2 tokens
- AdsCenterGo：按操作类型消耗

#### Token 充值包
- 小包：¥99 = 10,000 tokens
- 中包：¥299 = 50,000 tokens (40% off)
- 大包：¥599 = 200,000 tokens (67% off)
- 超大包：¥999 = 500,000 tokens (80% off)

### 2.8 通用功能
- 响应式设计（支持移动端）
- 深色/浅色主题切换
- 多语言支持（中文/英文）
- 实时通知系统
- 数据导出功能（CSV/Excel）

## 4. 技术架构

### 4.1 整体架构
```
Frontend (Next.js) ←→ API Layer (Go + GoFly) ←→ Database (MySQL 8.0)
                     ↑
                GoFly Admin V3
```

### 4.2 数据库设计
使用新数据库，无需数据迁移。核心表包括：
- users（用户表）
- plans（套餐表）
- subscriptions（订阅表）
- token_transactions（Token 交易表）
- batchgo_tasks（BatchGo 任务表）
- siterankgo_queries（SiteRankGo 查询表）
- adscentergo_accounts（AdsCenterGo 账户表）

### 4.3 API 设计
- 用户 API：`/api/v1/*`
- 管理员 API：`/admin/api/*`
- 统一响应格式：`{code, message, data, pagination?}`
- 支持长任务处理（创建任务→查询进度→获取结果）

### 4.4 认证授权
- 前端用户：JWT + OAuth2.0
- 管理员：独立账号密码系统
- RBAC 权限模型
- 基于套餐的功能限制
